# ✅ 上线前必做清单

## 🔴 必须完成（否则网站无法正常工作）

### 1. 配置部署平台环境变量 ⚠️ 最重要！

**如果不配置，网站无法连接 Supabase，所有数据功能都会失效！**

#### Vercel
1. 登录 Vercel Dashboard
2. 选择你的项目
3. 进入 `Settings` → `Environment Variables`
4. 添加以下两个变量：
   ```
   VITE_SUPABASE_URL = https://你的项目.supabase.co
   VITE_SUPABASE_ANON_KEY = 你的 anon key
   ```
5. 选择环境：`Production`, `Preview`, `Development`（全选）
6. 点击 `Save`
7. **重要**：重新部署（Settings → Deployments → 选择最新部署 → Redeploy）

#### Netlify
1. 登录 Netlify Dashboard
2. 选择你的站点
3. 进入 `Site settings` → `Environment variables`
4. 添加相同的两个变量
5. 点击 `Save`
6. **重要**：重新部署（Deploys → Trigger deploy）

#### Railway / Render / 其他平台
在平台的环境变量设置中添加相同的两个变量，然后重新部署。

**如何获取这些值**：
- 打开 `.env.local` 文件（本地开发用的）
- 复制 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY` 的值
- 或者从 Supabase Dashboard → Settings → API 获取

## ⚠️ 常见问题

### Q: 部署后网站显示空白？
**A**: 
1. 检查环境变量是否配置
2. 检查浏览器控制台错误（F12）
3. 确认 Supabase 项目正常运行

### Q: Admin 页面无法登录？
**A**: 
1. 确认 Supabase Auth 用户已创建
2. 确认邮箱和密码正确
3. 检查 Supabase Dashboard → Authentication → Users

### Q: 数据无法保存？
**A**: 
1. 检查浏览器控制台错误
2. 检查环境变量是否正确配置
3. 检查 Supabase Dashboard → Database → Logs

### Q: 图片上传失败？
**A**: 
1. 检查 Supabase Storage bucket 是否创建
2. 检查 Storage RLS 策略
3. 检查文件大小（不超过限制）
