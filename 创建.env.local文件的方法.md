# 如何创建 .env.local 文件

## 🎯 最简单的方法（推荐）

### 方法 1：在 Cursor 中直接创建

1. **在 Cursor 中，按 `Ctrl+N` 创建新文件**

2. **粘贴以下内容：**
   ```env
   VITE_SUPABASE_URL=https://your-project-id.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-key-here
   ```

3. **保存文件：**
   - 按 `Ctrl+S` 打开保存对话框
   - **重要**：文件名输入 `.env.local`（注意前面的点）
   - 保存位置选择项目根目录（与 `package.json` 同级）

4. **如果提示文件名无效：**
   - 先保存为 `env.local`（不带点）
   - 然后在文件管理器中重命名为 `.env.local`

### 方法 2：使用文件管理器创建

1. **打开文件管理器（Windows 资源管理器）**

2. **导航到项目文件夹：**
   ```
   C:\Users\吴昊天\Desktop\MSBC网站开发\msbc-website
   ```

3. **创建新文件：**
   - 在空白处右键点击
   - 选择 "新建" → "文本文档"
   - 命名为 `.env.local`（注意前面的点）
   - 如果系统提示"必须输入文件名"，先命名为 `env.local`，然后重命名

4. **编辑文件：**
   - 右键点击文件 → 打开方式 → 记事本
   - 粘贴以下内容：
     ```env
     VITE_SUPABASE_URL=https://your-project-id.supabase.co
     VITE_SUPABASE_ANON_KEY=your-anon-key-here
     ```
   - 保存（Ctrl+S）

### 方法 3：复制模板文件

1. **在项目根目录找到 `env.local.template` 文件**

2. **复制这个文件**

3. **重命名为 `.env.local`**

4. **打开并编辑，填入你的 Supabase 信息**

## 📝 填写内容

打开 `.env.local` 文件后，替换以下内容：

### 1. 替换 Supabase URL

找到这一行：
```
VITE_SUPABASE_URL=https://your-project-id.supabase.co
```

替换为你的实际 URL（从 Supabase Dashboard 复制）：
```
VITE_SUPABASE_URL=https://你的项目ID.supabase.co
```

### 2. 替换 anon key

找到这一行：
```
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

替换为你的实际 anon key（从 Supabase Dashboard 复制）：
```
VITE_SUPABASE_ANON_KEY=你的完整anon密钥
```

## ✅ 验证文件创建成功

创建完成后，在 Cursor 中：

1. **按 `Ctrl+P` 打开快速文件搜索**
2. **输入 `.env.local`**
3. **如果文件存在，会显示出来**
4. **双击打开，检查内容是否正确**

## 🆘 如果还是看不到文件

### 在 Cursor 中显示隐藏文件：

1. **打开设置**：`Ctrl+,`
2. **搜索 "files.exclude"**
3. **找到 `.env*` 相关的排除规则**
4. **删除或注释掉这些规则**

### 或者使用命令行验证：

在终端中运行：
```powershell
Get-ChildItem -Force | Where-Object { $_.Name -like ".env*" }
```

如果看到 `.env.local`，说明文件已创建成功。
