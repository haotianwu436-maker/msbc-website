# 🚀 MSBC 网站快速测试指南

## 📋 测试前准备

1. **打开网站**：访问你的 Vercel 部署地址
2. **打开开发者工具**：按 `F12`，查看 `Console` 标签
3. **准备测试账号**：Supabase Admin 账号（用于测试 Admin 页面）

---

## ✅ 核心功能测试（必测）

### 1. 首页测试（5分钟）

访问：`你的网站域名/2026`

**检查项：**
- [ ] 页面正常加载，没有白屏
- [ ] Hero 区域显示正常（标题、日期、按钮）
- [ ] 导航栏正常显示（About, Agenda, Speakers, Hackathon, FAQ）
- [ ] 统计信息条正常显示
- [ ] Speakers Preview 显示演讲者卡片（至少 3-4 个）
- [ ] Agenda Preview 显示议程列表
- [ ] Sponsors Preview 显示赞助商 Logo
- [ ] FAQ Preview 显示常见问题
- [ ] Contact 区域在页面底部正常显示
- [ ] Footer 正常显示

**测试链接：**
- [ ] 点击 "See More Speakers" → 跳转到 Speakers 页面
- [ ] 点击 "View Full Agenda" → 跳转到 Agenda 页面
- [ ] 点击 "View All Sponsors" → 跳转到 Sponsors 页面
- [ ] 点击 "View All FAQs" → 跳转到 FAQ 页面

---

### 2. Speakers 页面测试（3分钟）

访问：`你的网站域名/2026/speakers`

**检查项：**
- [ ] 页面正常加载
- [ ] 显示演讲者列表
- [ ] 每个演讲者卡片包含：头像、姓名、职位、公司
- [ ] Featured Speakers 区域正常显示
- [ ] 其他演讲者区域正常显示
- [ ] 悬停效果正常（卡片有交互反馈）
- [ ] 响应式布局正常（缩小浏览器窗口测试）

---

### 3. Agenda 页面测试（3分钟）

访问：`你的网站域名/2026/agenda`

**检查项：**
- [ ] 页面正常加载
- [ ] 显示议程列表
- [ ] 日期分组正确（8.29 和 8.30）
- [ ] 每个议程项显示：时间、标题、描述、演讲者
- [ ] 布局美观，易读

---

### 4. Sponsors 页面测试（2分钟）

访问：`你的网站域名/2026/sponsors`

**检查项：**
- [ ] 页面正常加载
- [ ] 显示赞助商 Logo
- [ ] Logo 图片清晰
- [ ] 布局合理（如果有分类，分类正常）

---

### 5. FAQ 页面测试（2分钟）

访问：`你的网站域名/2026/faq`

**检查项：**
- [ ] 页面正常加载
- [ ] 显示常见问题列表
- [ ] 点击问题可以展开/折叠答案
- [ ] 交互流畅

---

### 6. Admin 页面测试（10分钟）⭐ 重点

访问：`你的网站域名/admin`

#### 6.1 登录测试
- [ ] 显示登录页面
- [ ] 使用 Supabase 账号登录（邮箱 + 密码）
- [ ] 登录成功后进入管理界面
- [ ] 错误密码显示错误提示

#### 6.2 Speakers 管理测试
- [ ] 点击左侧菜单 "Speakers"
- [ ] 显示现有演讲者列表
- [ ] 点击 "Add Speaker" 添加新演讲者
- [ ] 填写信息：姓名、职位、公司、简介
- [ ] 上传头像图片
- [ ] 保存后演讲者出现在列表中
- [ ] 点击编辑按钮可以修改信息
- [ ] 修改后自动保存（防抖功能，延迟 2 秒）
- [ ] 点击删除按钮可以删除演讲者
- [ ] 删除后列表更新

#### 6.3 数据同步测试
- [ ] 在 Admin 页面添加/修改一个演讲者
- [ ] 打开新标签页，访问首页
- [ ] 刷新页面（F5）
- [ ] 检查首页的 Speakers Preview 是否显示新添加的演讲者
- [ ] 访问 Speakers 页面，确认新演讲者出现

#### 6.4 其他管理模块测试（快速检查）
- [ ] Site Settings 可以编辑
- [ ] Agenda 可以添加/编辑/删除
- [ ] Sponsors 可以添加/编辑/删除
- [ ] FAQ 可以添加/编辑/删除
- [ ] 每个模块保存后数据持久化

---

### 7. 导航菜单测试（3分钟）

**测试所有导航链接：**
- [ ] 点击 "About" → 跳转到首页的 About 区域
- [ ] 点击 "Agenda" → 跳转到 Agenda 页面
- [ ] 点击 "Speakers" → 跳转到 Speakers 页面
- [ ] 点击 "Hackathon" → 跳转到 Hackathon 页面
- [ ] 点击 "FAQ" → 跳转到 FAQ 页面
- [ ] 点击 "Register" → 跳转到 Tickets 页面
- [ ] 点击 Logo → 返回首页

---

### 8. 响应式设计测试（5分钟）

**测试不同屏幕尺寸：**

1. **桌面端（1920x1080）**
   - [ ] 布局正常，内容居中
   - [ ] 文字大小合适
   - [ ] 图片显示正常

2. **平板端（768px）**
   - [ ] 打开浏览器开发者工具（F12）
   - [ ] 点击设备工具栏图标（或按 Ctrl+Shift+M）
   - [ ] 选择 iPad 尺寸
   - [ ] 检查布局是否自适应
   - [ ] 导航菜单是否正常

3. **移动端（375px）**
   - [ ] 选择 iPhone 尺寸
   - [ ] 检查汉堡菜单是否显示
   - [ ] 点击汉堡菜单可以打开/关闭
   - [ ] 文字大小是否合适
   - [ ] 按钮大小是否合适
   - [ ] 图片是否正常显示

---

### 9. 浏览器控制台检查（2分钟）

**检查错误：**
1. 按 `F12` 打开开发者工具
2. 点击 `Console` 标签
3. 检查是否有红色错误（除了第三方扩展的警告）
4. 确认没有 Supabase 连接错误

**应该看到的（正常）：**
- ✅ 可能有一些第三方扩展的警告（TronWeb、umami 等），这些可以忽略
- ✅ 没有 `supabaseUrl is required` 错误
- ✅ 没有 `Failed to fetch` 错误

---

### 10. 性能测试（2分钟）

**检查加载速度：**
- [ ] 打开浏览器开发者工具（F12）
- [ ] 点击 `Network` 标签
- [ ] 刷新页面（F5）
- [ ] 检查页面加载时间（应该在 3 秒内）
- [ ] 检查所有资源是否加载成功（200 状态码）
- [ ] 没有 404 错误

---
