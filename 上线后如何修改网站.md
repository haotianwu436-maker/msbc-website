# 📝 上线后如何修改网站

## 🎯 两种修改方式

### 方式 1: 通过 Admin 页面修改内容（无需代码）✅

**可以修改的内容**（通过 `/admin` 页面）：
- ✅ Speakers（演讲者信息、照片）
- ✅ Sponsors（赞助商信息、Logo）
- ✅ Agenda（日程安排）
- ✅ FAQ（常见问题）
- ✅ Universities（大学信息）
- ✅ Contact（联系方式）
- ✅ 首页内容（文本、设置）
- ✅ 图片上传和替换

**特点**：
- ⚡ **即时生效**：修改后立即保存到数据库
- 👥 **非技术人员可用**：不需要懂代码
- 🔄 **无需重新部署**：修改后刷新网站即可看到

**操作流程**：
1. 访问 `https://你的域名.com/admin`
2. 登录
3. 修改内容
4. 保存
5. 刷新网站，立即看到变化

---

### 方式 2: 修改代码（UI、样式、功能）💻

**可以修改的内容**（需要修改代码）：
- 🎨 UI 设计（颜色、布局、样式）
- 📱 页面结构（添加新页面、修改布局）
- ⚙️ 功能（添加新功能、修改交互）
- 🎭 动画效果
- 📐 响应式布局

**特点**：
- 🔧 **需要懂代码**：需要修改 React/TypeScript 代码
- 🚀 **需要重新部署**：修改后需要重新构建和部署
- ⏱️ **需要时间**：通常几分钟到几十分钟

**操作流程**：
1. 在本地修改代码（使用 Cursor/VS Code）
2. 测试本地效果（`pnpm dev`）
3. 提交代码到 GitHub（`git add`, `git commit`, `git push`）
4. 部署平台自动重新部署（Vercel/Netlify）
5. 等待部署完成（通常 1-5 分钟）
6. 刷新网站，看到新变化

---

## 🎨 常见修改场景

### 场景 1: 添加新的 Speaker
**方式**: Admin 页面 ✅
- 访问 `/admin` → Speakers → Add
- 填写信息，上传照片
- 保存，立即生效

### 场景 2: 修改网站颜色
**方式**: 修改代码 💻
- 修改 `client/src/index.css` 中的 CSS 变量
- 或修改 Tailwind 配置
- 提交代码，重新部署

### 场景 3: 修改首页标题文字
**方式**: Admin 页面 ✅（如果标题在 Site Settings 中）
**或**: 修改代码 💻（如果标题是硬编码的）

### 场景 4: 添加新页面
**方式**: 修改代码 💻
- 创建新页面组件（如 `client/src/pages/NewPage.tsx`）
- 添加路由（在 `client/src/App.tsx`）
- 提交代码，重新部署

### 场景 5: 修改按钮样式
**方式**: 修改代码 💻
- 修改组件的 CSS/Tailwind 类
- 提交代码，重新部署

## 🔄 修改代码的详细流程

### 步骤 1: 本地修改
```bash
# 1. 打开项目
cd msbc-website

# 2. 启动开发服务器
pnpm dev

# 3. 在浏览器查看效果（http://localhost:3000）
# 4. 修改代码，实时看到变化
```

### 步骤 2: 测试
- 在本地测试所有功能
- 确认修改正确
- 检查移动端显示

### 步骤 3: 提交代码
```bash
# 1. 查看修改
git status

# 2. 添加修改
git add .

# 3. 提交
git commit -m "修改 UI: 更新按钮样式"

# 4. 推送到 GitHub
git push
```

### 步骤 4: 自动部署
- **Vercel/Netlify**: 自动检测 GitHub 推送，开始部署
- 等待部署完成（通常 1-5 分钟）
- 部署完成后，网站自动更新

## ⚠️ 注意事项

### 1. 备份重要修改
修改代码前，建议：
- 提交到 GitHub（版本控制）
- 或创建备份分支

### 2. 测试后再部署
- 先在本地测试
- 确认无误后再推送代码

### 3. 部署时间
- 通常 1-5 分钟
- 部署期间网站可能短暂不可用（Vercel/Netlify 通常无感）

### 4. 回滚
如果部署后发现问题：
- **Vercel**: Dashboard → Deployments → 选择之前的版本 → Redeploy
- **Netlify**: Site overview → Deploys → 选择之前的版本 → Publish deploy
- **Git**: `git revert` 然后重新推送
