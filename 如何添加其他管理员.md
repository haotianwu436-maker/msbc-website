# 如何添加其他管理员账号

## 📋 方案概述

有两种方式让其他小伙伴也能修改数据：

### 方案 1: Supabase Auth（推荐）✅
- 每个小伙伴有独立的邮箱和密码
- 可以追踪谁做了什么操作
- 更安全，可以单独撤销某个人的权限
- 支持密码重置

### 方案 2: 共享简单密码（不推荐）
- 所有人用同一个密码
- 无法追踪操作者
- 安全性较低

---

## 🚀 方案 1: 使用 Supabase Auth 添加管理员（推荐）

### 步骤 1: 在 Supabase Dashboard 创建用户

1. **登录 Supabase Dashboard**
   - 访问：https://app.supabase.com
   - 选择你的项目

2. **进入 Authentication 页面**
   - 左侧菜单 → `Authentication`
   - 点击 `Users` 标签

3. **创建新用户**
   - 点击右上角 `Add user` 或 `Invite user`
   - 选择 `Create new user`
   - 填写信息：
     ```
     Email: 小伙伴的邮箱（例如：xiaoming@example.com）
     Password: 设置一个临时密码（建议至少 12 位）
     Auto Confirm User: ✅ 勾选（自动确认，无需邮箱验证）
     ```
   - 点击 `Create user`

4. **发送账号信息给小伙伴**
   - 邮箱：`xiaoming@example.com`
   - 密码：你设置的临时密码
   - 登录地址：`https://你的域名.com/admin`

5. **让小伙伴首次登录后修改密码**
   - 登录后，可以引导他们修改密码（需要添加密码修改功能，或手动在 Supabase Dashboard 修改）

### 步骤 2: 更新 RLS 策略（确保只有认证用户能写入）

在 Supabase SQL Editor 中执行以下 SQL：

```sql
-- 更新 Speakers 表的 RLS 策略
DROP POLICY IF EXISTS "Allow public write access" ON speakers;
CREATE POLICY "Allow authenticated write access" ON speakers
  FOR ALL USING (auth.role() = 'authenticated');

-- 更新 Sponsors 表的 RLS 策略
DROP POLICY IF EXISTS "Allow public write access" ON sponsors;
CREATE POLICY "Allow authenticated write access" ON sponsors
  FOR ALL USING (auth.role() = 'authenticated');

-- 更新 Agenda 表的 RLS 策略
DROP POLICY IF EXISTS "Allow public write access" ON agenda_items;
CREATE POLICY "Allow authenticated write access" ON agenda_items
  FOR ALL USING (auth.role() = 'authenticated');

-- 更新 FAQ 表的 RLS 策略
DROP POLICY IF EXISTS "Allow public write access" ON faq_items;
CREATE POLICY "Allow authenticated write access" ON faq_items
  FOR ALL USING (auth.role() = 'authenticated');

-- 更新 Storage 权限（如果需要）
-- Storage 的 RLS 策略已经在创建 bucket 时设置，确保只有认证用户可以上传
```

### 步骤 3: 测试登录

1. 让小伙伴访问：`https://你的域名.com/admin`
2. 输入邮箱和密码
3. 确认可以正常登录和修改数据

## 🔐 用户管理最佳实践

### 1. 创建用户时
- ✅ 使用工作邮箱（不要用个人邮箱）
- ✅ 设置强密码（至少 12 位）
- ✅ 勾选 "Auto Confirm User"（避免邮箱验证问题）

### 2. 密码管理
- ✅ 首次登录后，建议修改密码
- ✅ 定期更换密码（每 3-6 个月）
- ✅ 不要共享密码

### 3. 权限管理
- ✅ 只给需要的人创建账号
- ✅ 如果有人离职，及时删除账号
- ✅ 定期检查用户列表

### 4. 操作追踪（可选）
如果需要追踪谁做了什么操作，可以：
- 在数据库表中添加 `created_by` 和 `updated_by` 字段
- 记录操作日志

## 🗑️ 删除用户账号

如果需要删除某个用户的访问权限：

1. **在 Supabase Dashboard**
   - `Authentication` → `Users`
   - 找到要删除的用户
   - 点击 `...` → `Delete user`

2. **注意事项**
   - 删除后，该用户无法再登录
   - 不会删除他们创建的数据（数据会保留）

---

## 🔄 重置密码

如果用户忘记密码：

### 方法 1: 在 Supabase Dashboard 重置
1. `Authentication` → `Users`
2. 找到用户 → 点击 `...` → `Reset password`
3. 输入新密码 → 保存
4. 告诉用户新密码

### 方法 2: 让用户自己重置（需要配置）
- 需要在登录页面添加 "忘记密码" 功能
- 会发送重置邮件到用户邮箱

## ❓ 常见问题

### Q: 用户登录时提示 "Invalid login credentials"？
**A**: 
1. 检查邮箱是否正确
2. 检查密码是否正确
3. 检查用户是否已确认（Auto Confirm User 是否勾选）

### Q: 用户登录后无法保存数据？
**A**: 
1. 检查 RLS 策略是否已更新（执行步骤 2 的 SQL）
2. 检查用户是否已认证（在 Supabase Dashboard 查看用户状态）

### Q: 如何批量创建用户？
**A**: 
- 建议使用 Supabase Dashboard 手动创建
- 或者使用 Supabase Admin API（需要编写脚本）

### Q: 用户可以在 Admin 页面修改密码吗？
**A**: 
- 目前不支持，需要手动在 Supabase Dashboard 修改
- 后续可以添加密码修改功能
