## ⚠️ 需要注意的事项

### 1. Map 组件使用外部 API Key

**文件**: `client/src/components/Map.tsx`

**发现**:
```typescript
const API_KEY = import.meta.env.VITE_FRONTEND_FORGE_API_KEY;
```

**说明**:
- 如果使用 Map 组件，需要在部署平台配置 `VITE_FRONTEND_FORGE_API_KEY`
- 如果不使用 Map 组件，可以忽略

**建议**:
- 如果使用：在部署平台添加环境变量
- 如果不使用：可以忽略，不影响其他功能

### 2. Analytics 配置（可选）

**文件**: `client/index.html`

**发现**:
```html
<script
  defer
  src="%VITE_ANALYTICS_ENDPOINT%/umami"
  data-website-id="%VITE_ANALYTICS_WEBSITE_ID%"></script>
```

**说明**:
- 使用了占位符，如果没有配置环境变量，脚本不会加载
- 这是可选的，不影响网站功能

**建议**:
- 如果需要 Analytics：配置环境变量
- 如果不需要：可以忽略

## 🎯 上线前最后步骤

### 步骤 1: 配置环境变量（必须）⚠️

**在部署平台配置**:
1. 登录部署平台（Vercel/Netlify）
2. 进入项目设置 → Environment Variables
3. 添加：
   ```
   VITE_SUPABASE_URL = （从 .env.local 复制）
   VITE_SUPABASE_ANON_KEY = （从 .env.local 复制）
   ```

4. 选择所有环境（Production, Preview, Development）
5. 保存

**配置环境变量后，必须重新部署才能生效**:
- Vercel: Settings → Deployments → 选择最新部署 → Redeploy

## 📞 如果遇到问题

### 部署后网站无法连接 Supabase
1. 检查环境变量是否配置
2. 检查环境变量值是否正确
3. 检查是否重新部署了
4. 检查浏览器控制台错误（F12）

### Admin 页面无法登录
1. 确认 Supabase Auth 用户已创建
2. 确认邮箱和密码正确
3. 检查 Supabase Dashboard → Authentication → Users

### 数据无法保存
1. 检查浏览器控制台错误
2. 检查 Supabase Dashboard → Database → Logs
3. 确认 RLS 策略正确配置
