## 📋 上线前最后检查清单

### 1. 部署平台环境变量配置 ⚠️

**必须在部署平台配置环境变量**，否则网站无法连接 Supabase：

#### Vercel
1. 项目设置 → `Environment Variables`
2. 添加：
   ```
   VITE_SUPABASE_URL = 你的 Supabase 项目 URL
   VITE_SUPABASE_ANON_KEY = 你的 Supabase Anon Key
   ```
3. 选择环境：`Production`, `Preview`, `Development`（全选）
4. 点击 `Save`
5. 重新部署

#### Netlify
1. Site settings → `Environment variables`
2. 添加相同的变量
3. 重新部署

#### Railway / Render / 其他平台
在平台的环境变量设置中添加相同的变量

## ⚠️ 重要提醒

### 环境变量必须配置
如果部署后网站无法连接 Supabase，通常是环境变量未配置导致的。

**检查方法**：
1. 打开浏览器控制台（F12）
2. 查看是否有 Supabase 连接错误
3. 如果有，检查环境变量是否正确配置

### Admin 页面访问
部署后，Admin 页面访问地址：
```
https://你的域名.com/admin
```

### 数据备份
- Supabase 会自动备份数据库
- 建议定期导出数据作为额外备份

## 📞 遇到问题？

### 部署后无法访问 Supabase
- 检查环境变量是否配置
- 检查 Supabase 项目是否正常运行
- 检查浏览器控制台错误信息

### Admin 页面无法登录
- 确认 Supabase Auth 用户已创建
- 确认邮箱和密码正确
- 检查 Supabase Dashboard → Authentication → Users

### 数据无法保存
- 检查浏览器控制台错误
- 检查 Supabase Dashboard → Database → Logs
- 确认 RLS 策略正确配置
