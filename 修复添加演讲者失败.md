# 修复添加演讲者失败的问题

## 问题原因

Supabase 的 RLS (Row Level Security) 策略要求用户必须是**已认证用户**才能写入数据，但 Admin 页面目前没有使用 Supabase Auth 进行认证。

## 解决方案

我已经更新了 `supabase/migrations/001_initial_schema.sql` 文件，将写入策略从要求认证改为允许公开写入。

## 操作步骤

### 方法 1: 在 Supabase Dashboard 中手动更新策略（推荐）

1. 打开 Supabase Dashboard: https://supabase.com/dashboard
2. 选择你的项目
3. 进入 **Authentication** > **Policies**
4. 或者进入 **Table Editor** > 选择 `speakers` 表 > 点击 **Policies** 标签

5. 对于 `speakers` 表：
   - 找到策略 "Allow authenticated write access"
   - 点击 **Edit** 或 **Delete**
   - 点击 **New Policy**
   - 策略名称: `Allow public write access`
   - 策略类型: `For all operations`
   - 策略定义: `USING (true)`
   - 点击 **Save**

6. 重复步骤 5，更新以下表：
   - `sponsors`
   - `agenda_items`
   - `faq_items`

### 方法 2: 使用 SQL Editor 直接执行

1. 打开 Supabase Dashboard
2. 进入 **SQL Editor**
3. 执行以下 SQL：

```sql
-- 删除旧的策略
DROP POLICY IF EXISTS "Allow authenticated write access" ON speakers;
DROP POLICY IF EXISTS "Allow authenticated write access" ON sponsors;
DROP POLICY IF EXISTS "Allow authenticated write access" ON agenda_items;
DROP POLICY IF EXISTS "Allow authenticated write access" ON faq_items;

-- 创建新的公开写入策略
CREATE POLICY "Allow public write access" ON speakers FOR ALL USING (true);
CREATE POLICY "Allow public write access" ON sponsors FOR ALL USING (true);
CREATE POLICY "Allow public write access" ON agenda_items FOR ALL USING (true);
CREATE POLICY "Allow public write access" ON faq_items FOR ALL USING (true);
```

4. 点击 **Run** 执行

## 验证修复

1. 刷新 Admin 页面
2. 点击 "Speakers" 菜单
3. 点击 "Add Speaker"
4. 应该能成功创建新的演讲者

## 安全提示

⚠️ **注意**: 当前策略允许任何人写入数据。在生产环境中，你应该：

1. 使用 Supabase Auth 进行用户认证
2. 只允许已认证的管理员写入数据
3. 或者使用 Service Role Key（仅服务器端使用）

## 如果还是失败

请检查：

1. **浏览器控制台** (F12) 是否有错误信息
2. **Supabase Dashboard** > **Logs** 查看是否有数据库错误
3. **.env.local** 文件中的 Supabase URL 和 Key 是否正确
4. 数据库表是否已正确创建（检查 Table Editor）
